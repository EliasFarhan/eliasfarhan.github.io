**GPR5300 Editor** Version 1.00 Welcome to the API documentation of the GPR5300 - Computer Graphics editor for Game Art students at [SAE Institute Geneva](https://geneva.sae.edu/). You can download it here: https://github.com/SAE-Geneve/GPR5300_920_Editor. Context ============================================================== During the Computer Graphics module at SAE Institute Geneva, the Games Programming students have to create their own OpenGL rendering engine in C++. Starting 2022, some game art's students started to follow the course to get some technical background on the graphics pipeline. So I created this graphics editor that allow to serialize a pipeline into a scene package that can be played into the player executable. With this tool, the game art's can implement 3d scene using basic scripting in Python and control the objects of the pipeline, without hard knowledge in programming. Features ============================================================== The GPR5300 Editor features: - GLSL shaders (vertex, fragment, geometry and compute shaders) - Python scripting - Serialization of pipelines, materials, render passes and framebuffers. Requirement for the editor -------------------------------------------------------------- - [Python 3.8](https://www.python.org/downloads/release/python-380/) - [Visual Studio 2022 Redistributable](https://docs.microsoft.com/en-us/visualstudio/releases/2022/redistribution) - [Vulkan SDK](https://vulkan.lunarg.com/) Graphics Objects =============================================================== Shaders --------------------------------------------------------------- When loading a shader take care of using the correct extension: - Vertex shaders: .vert - Fragment shaders: .frag - Geometry shaders: .geom - Compute shaders: .comp Pipeline --------------------------------------------------------------- Pipeline serialized data file format is .pipe. Textures --------------------------------------------------------------- The following formats are supported to load textures (.jpg, .png, .tga, .bmp, .hdr). For cubemap, the path to the textures are stored in a .cube file.
